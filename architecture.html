Architecture

LIVRE OS is organized as a layered operating system: from COTI's gcEVM to Solivre, LivreID, LivreVault, and the applications that consume proofs.

Layer Model

  • Layer 5 - Apps & Institutions
  • Layer 4 - LivreKit SDK & APIs
  • Layer 3 - LivreID, Proof Engine, LivreVault
  • Layer 2 - Solivre Kernel
  • Layer 1 - Core Contracts on COTI
  • Layer 0 - COTI gcEVM Network

Design Principles

  • Zero raw identity on-chain
  • User-owned keys and storage
  • Minimal, non-linkable proofs
  • Composable claims and policies
  • Auditable contracts & circuits

Full architecture specifications live in the internal LIVRE OS documentation (Solivre Kernel, LivreID, Proof Engine, Vault, Key Management, Nodes & Compute).
